移动端适配：三种方案

在已有基础上改进 vs 重新设计

A

渐进式增强

最小改动路线

  • 增大触摸区域（节点 44px 触控范围）
  • 模态框响应式（最大宽度适配屏幕）
  • 刻度自适应间隔
  • 保持现有侧边栏折叠模式

优点

改动小、风险低、1-2 小时完成

缺点

表单体验没改善、加事件仍需 4 步
B

底部面板 + 浮动按钮

表单体验优先

  • 点击浮动按钮 → 底部滑出添加面板
  • 面板占下半屏，不离开时间轴视图
  • 事件节点触控面积增大到 44px
  • 模态框变为底部弹出式（Bottom Sheet）

优点

加事件只需 2 步、不跳转页面、沉浸感强

缺点

需要新增 CSS + HTML 结构，约 3-4 小时
C

底部 Tab 导航

完整重构移动端布局

  • 底部 Tab 栏：时间轴 | 添加 | 统计
  • 添加页为独立全屏视图
  • 编辑事件为底部弹出面板
  • 触摸全面优化 + 手势支持

优点

最接近原生 App 体验

缺点

改动量大（HTML/CSS/JS 都要改）、可能需要状态管理调整
